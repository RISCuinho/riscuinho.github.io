Lattice Mini FPGA Core Board Development Board ICE40UL1K-CM36A (CDPGA-Bx)

CDCTL_BX_E is the smallest system board of Lattice’s iCE40 UL series FPGA (ICE40UL1K-CM36A), (it is the CDCTL_BX bus controller unprogrammed version, the suffix E stands for Empty), on-board resources include: -Lead out 12 IO ports (4 of which are multiplexed with configuration ports); -16 M clock, higher frequency can be output via internal PLL; -It can be used with only 3.3 V external power supply.

Lattice devices are more convenient to use:

-Low price, small size, and many packaging options; -It can be internally configured to prevent cracking; -No private burning tool is needed, just use any FT232H series USB to SPI tool; -Development software and burning tools are free and relatively compact, and support Linux and Windows platforms (Linux version development software is less than 400 MByte).

The module supports 3 configuration methods:

-The SRAM is written by an external MCU or a burning tool, and the official SPI writing timing and sample code are provided by the FPGA. This method does not limit the number of writes. You can also switch the firmware at any time, but the firmware will be lost if power is off; -Provided by the external SPI FLASH configuration chip, FPGA is the master, and the firmware is automatically read after power-on (FLASH data can be written directly by the burning tool); -Programming the FPGA internal NVCM OTP memory can only be programmed once. After programming, it can be configured by the internal NVCM by default after power-on; and it can still be configured through the MCU, but the external SPI FLASH configuration method can no longer be used.
